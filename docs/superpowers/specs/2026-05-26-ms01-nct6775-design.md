# Minisforum MS-01 NCT6775 Support Design

## Background

Minisforum MS-01 reports DMI data as:

- System vendor: `Micro Computer (HK) Tech Limited`
- Product name: `Venus Series`
- Board vendor: `Shenzhen Meigao Electronic Equipment Co.,Ltd`
- Board name: `AHWSA`
- BIOS version: `AHWSA.1.22`

Initial Linux hwmon output did not expose fan RPM. `sensors-detect --auto`
found a supported Super I/O sensor chip:

```text
Found `Nuvoton NCT6798D Super IO Sensors'
    (address 0xa20, driver `nct6775')
```

Loading the in-tree Linux `nct6775` driver exposes:

```text
nct6798-isa-0a20
fan1: 2149 RPM
fan2: 1781 RPM
fan3: 0 RPM
fan4: 0 RPM
fan5: 0 RPM
fan7: 0 RPM
```

## Decision

Do not add MS-01 as a `platform_fans_hwmon` platform implementation.

MS-01 is already supported by the existing Linux mainline `nct6775` hwmon
driver. Reimplementing NCT6798D register access in this project would duplicate
a maintained kernel driver, increase maintenance cost, and risk exposing
duplicate hwmon devices for the same physical sensors.

Document MS-01 as an externally supported platform instead.

## Documentation Changes

Update project documentation to recommend this discovery flow before adding a
new platform:

1. Run `sudo sensors-detect --auto`.
2. Load any existing recommended kernel hwmon driver.
3. Check `sensors` and `/sys/class/hwmon`.
4. Add a `platform_fans_hwmon` backend/platform only when existing mainline
   drivers cannot expose the required fan RPM values.

Add MS-01 to the supported platform documentation as:

- Platform: Minisforum MS-01 / Venus Series / board `AHWSA`
- Chip: Nuvoton `NCT6798D`
- Driver: `nct6775`
- hwmon name: `nct6798`
- Adapter: `nct6798-isa-0a20`
- Valid fan RPM channels: `fan1_input`, `fan2_input`
- Zero RPM channels observed: `fan3_input`, `fan4_input`, `fan5_input`,
  `fan7_input`

Suggested persistent load command:

```bash
echo nct6775 | sudo tee /etc/modules-load.d/nct6775.conf
```

## Non-Goals

- Do not add a Super I/O backend for NCT6798D in this project.
- Do not write PWM configuration or fan-control registers.
- Do not label MS-01 as requiring `platform_fans_hwmon`.

## Validation

Runtime validation was performed on `ms-01.timandes.net` with Ubuntu kernel
`6.17.0-22-generic`. The `nct6775` module loaded successfully and exposed
`fan1_input` and `fan2_input` as non-zero RPM values through standard hwmon.
