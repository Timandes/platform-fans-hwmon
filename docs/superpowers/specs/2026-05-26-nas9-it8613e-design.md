# NAS-9 IT8613E Fan RPM Design

## Context

NAS-9 is an AMD Ryzen 7 5800H system where the current kernel exposes CPU,
GPU, and NVMe temperature sensors, but no fan RPM sensors.

Observed runtime data:

- Host: `nas-9.timandes.net`
- Kernel: `6.18.18-trim`
- System vendor: `Default string`
- Product name: `DS2308`
- Board vendor: `Default string`
- Board name: `Dinson`
- BIOS version: `DS2308.A..002`
- CPU: `AMD Ryzen 7 5800H with Radeon Graphics`

Current hwmon devices:

- `k10temp`
- `amdgpu`
- `nvme`

`sensors-detect --auto` detected:

```text
Found `ITE IT8613E Super IO Sensors'                        Success!
    (address 0xa30, driver `to-be-written')
```

No module was recommended by `sensors-detect`.

## Goal

Determine the safest path to expose NAS-9 fan RPM readings without duplicating
generic Super I/O chip support in `platform_fans_hwmon` unless that is the only
practical path.

## Non-Goals

- Do not add PWM control.
- Do not write fan curves.
- Do not probe unknown I/O ranges from `platform_fans_hwmon`.
- Do not add a chip-wide IT8613E implementation to this project before
  validating whether `it87` can support it.

## Options

### Option 1: Validate IT8613E-capable it87 support

Find and validate an `it87` variant or upstream patch set that supports
IT8613E. Test it on NAS-9 and confirm whether it exposes `fan*_input` through
the normal Linux hwmon ABI.

Safety update: the known v4 upstream IT8613E patch series was withdrawn after
review because the IT8613E probe path could trigger a NULL pointer dereference.
The implementation plan must not load the withdrawn v4 patch series on NAS-9.
It must first identify a fixed IT8613E candidate, or stop before runtime module
loading.

This is the preferred path because IT8613E is a Super I/O monitoring chip. Chip
support belongs in a chip driver such as `it87` when available, while this
project should remain focused on platform-specific fan sources that existing
mainline hwmon drivers cannot expose.

Success criteria:

- The driver binds to IT8613E at ISA address `0xa30`.
- `/sys/class/hwmon/*/name` identifies the IT8613E-backed hwmon device.
- At least one nonzero `fan*_input` value appears under that hwmon device.
- `sensors` reports fan RPM values.
- Loading the driver does not expose or require fan-control writes for this
  project's read-only use case.

If successful, document NAS-9 as externally supported by the validated `it87`
driver instead of adding `platform_fans_hwmon` code.

### Option 2: Add a DS2308/IT8613E read-only platform backend

Add a tightly scoped `platform_fans_hwmon` backend only if Option 1 fails and
there is a reliable register map for reading fan RPM on this board.

The backend would match the NAS-9 platform explicitly:

- Product name: `DS2308`
- Board name: `Dinson`
- BIOS version family: `DS2308.A..002` or a deliberately reviewed compatible
  range
- Super I/O chip: `ITE IT8613E`
- ISA address: `0xa30`

This option must only expose read-only `fan*_input` and `fan*_label`
attributes. It must not implement voltage, temperature, PWM, or automatic chip
discovery.

This is a fallback because it would bring Super I/O register logic into a
platform project. That boundary is acceptable only if the implementation is
board-specific and the existing chip-driver path cannot work.

### Option 3: Document unsupported and wait for upstream

Record NAS-9 as detected but unsupported until IT8613E support lands in a
maintainable kernel driver.

This has the lowest runtime risk, but it does not solve fan RPM visibility.

## Recommended Design

Use Option 1 first.

The project should treat NAS-9 as an IT8613E validation task, not immediately
as a new `platform_fans_hwmon` backend. The next implementation plan should
focus on validating an IT8613E-capable `it87` driver on NAS-9. Only after that
fails should the project design a board-specific read-only backend.

If Option 1 succeeds, update documentation only:

- Add a NAS-9 entry to `docs/supported-platforms.md` as an externally supported
  platform.
- Add validation notes with DMI data, chip, address, driver source, and observed
  fan RPM attributes.
- Keep `platform_fans_hwmon` unchanged.

If Option 1 fails, create a separate design for Option 2 before writing code.
That follow-up design must include the exact fan tachometer registers and RPM
conversion rules.

## Testing And Validation

For Option 1, validation should run on NAS-9 only after a fixed IT8613E
candidate is selected:

```bash
sudo modprobe it87
for h in /sys/class/hwmon/hwmon*; do
  [ -e "$h/name" ] || continue
  echo "hwmon=$h name=$(cat "$h/name")"
  for f in "$h"/fan*_input "$h"/fan*_label; do
    [ -e "$f" ] && printf '%s=' "$f" && cat "$f"
  done
done
sensors
dmesg | grep -iE 'it87|it8613|hwmon' | tail -80
```

The result should be recorded before any persistent installation.
